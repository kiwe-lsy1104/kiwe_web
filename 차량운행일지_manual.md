# [매뉴얼] 법인 전기차 차량운행관리 시스템 (KiWE)

본 문서는 KiWE 웹 시스템의 **'차량운행일지'** 기능에 대한 사용 방법과 시스템의 동작 원리를 설명합니다.

---

## 1. 시스템 개요
본 시스템은 법인 차량(특히 전기차)의 운행 내역을 기록하고, 국세청 제출용 양식에 맞춰 엑셀로 출력하기 위한 관리 도구입니다. PC와 모바일 환경을 모두 지원하며, 실시간 데이터 동기화를 통해 현장에서 즉시 기록할 수 있습니다.

---

## 2. 주요 기능 및 사용 방법

### 1) 운행 기록 추가 (Driving Record)
- **PC**: 하단의 **[새 운행기록 추가]** 버튼을 클릭합니다.
- **모바일**: 상단의 파란색 **[운행기록추가]** 버튼을 클릭합니다.
- **동작**: 
    - 최신 기록의 '운행 후 계기판 거리'를 자동으로 가져와 '운행 전' 거리로 지정합니다.
    - 현재 날짜와 로그인한 사용자 실명이 자동으로 입력됩니다.
    - 운행 목적을 선택하거나 직접 입력한 뒤, 운행을 마친 후의 **계기판 숫자**를 입력하면 주행 거리가 자동 계산됩니다.

### 2) 충전 기록 추가 (Charging/Fueling Record)
- **PC**: 하단의 초록색 **[충전 기록 추가]** 버튼을 클릭합니다.
- **모바일**: 상단의 초록색 **[전기차 충전 기록]** 버튼을 클릭합니다.
- **동작**:
    - 운행 목적이 자동으로 **'충전'**으로 설정됩니다.
    - 주행 거리가 발생하지 않도록 **'운행 전/후' 거리가 동일하게** 세팅됩니다.
    - 사용자는 충전 금액과 충전량(kWh)만 입력하면 됩니다.

### 3) 엑셀 다운로드 (국세청 양식)
- **PC**: 하단의 **[국세청 양식 다운로드]** 버튼을 클릭합니다.
- 사용 중인 차량 정보와 해당 연도의 전체 데이터를 국세청 비업무용 승용차 운행기록부 표준 양식에 맞게 변형하여 내려받습니다.
- 목적지에 '출퇴근'이 포함된 경우 자동으로 출퇴근용 주행거리로 분류됩니다.

---

## 3. 기능별 동작 원리 (데이터 처리 로직)

### 1) 계기판 거리의 연속성 (Odometer Tracking)
- 본 시스템은 '단순 주행 거리'가 아닌 **'계기판 숫자(Odometer)'** 입력을 원칙으로 합니다.
- 새 행을 추가할 때 시스템은 `logs[0]`(가장 최근 데이터)의 '운행 후' 거리를 조회하여 `prevDist`로 사용합니다. 이를 통해 수동 입력 시 발생할 수 있는 오타와 거리 누락을 방지합니다.

### 2) 데이터 정렬 (Sorting Logic)
- **조회 시**: 날짜 역순(최신순)으로 정렬하여 현재의 차량 상태를 바로 파악할 수 있게 합니다.
- **저장 시**: 입력한 날짜와 생성 시간을 기준으로 데이터베이스(Supabase)에 안전하게 저장됩니다.

### 3) 통계 계산 (Auto Statistics)
- **당월 주행**: 현재 선택된 월의 모든 [(운행 후 - 운행 전)](file:///c:/Users/lee%20seung%20yong/OneDrive/%EB%B0%94%ED%83%95%20%ED%99%94%EB%A9%B4/kiwe_web/kiwe_web/js/analysis_list.js#172-176) 합계입니다.
- **전비(Efficiency)**: `총 주행거리 / 총 충전량`을 실시간 계산하여 차량의 에너지 효율을 보여줍니다.

---

## 4. 유의 사항 및 팁
- **실시간 저장**: 각 행(카드) 우측 또는 하단의 **[저장]** 버튼을 눌러야 데이터베이스에 최종 반영됩니다.
- **입력 실수 수정**: 잘못 입력된 숫자는 해당 칸을 클릭해 바로 수정하고 다시 저장할 수 있습니다.
- **모바일 홈 화면 추가**: 모바일 브라우저의 '홈 화면에 추가' 기능을 사용하면 앱처럼 편리하게 접근할 수 있습니다.
